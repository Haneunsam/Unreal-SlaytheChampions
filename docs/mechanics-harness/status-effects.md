# status-effects.md

## 목적

상태이상, 버프, 디버프가 카드 이벤트 큐 전투에서 어떤 순서로 적용되고 해제되는지 정의한다.

## 기본 분류

| 분류 | 설명 | 예시 |
| --- | --- | --- |
| Buff | 아군에게 이로운 효과 | 힘, 보호, 집중 |
| Debuff | 대상에게 불리한 효과 | 취약, 약화, 표식 |
| Control | 행동을 제한하는 효과 | 기절, 속박 |
| Trigger | 조건 충족 시 발동 | 반격, 연쇄, 복수 |
| Curse | 덱이나 행동을 방해 | 부상, 혼란, 과열 |
| Ultimate Stack | 필살기 조건을 쌓는 상태 | 강화, 집중, 분노 |

## 적용 타이밍

상태 효과는 아래 순서로 처리한다.

```text
이벤트 실행 시작
-> 이벤트 선처리 효과 확인
-> 피해/방어/주요 효과 처리
-> 이벤트 후처리 트리거 확인
-> 사망/전투 종료 확인
```

## 기본 상태이상 후보

| 이름 | 대상 | 효과 |
| --- | --- | --- |
| 취약 | 적/아군 | 받는 피해 증가 |
| 약화 | 적/아군 | 주는 피해 감소 |
| 표식 | 적 | 특정 카드나 파티원이 우선 공격하거나 추가 효과 발동 |
| 기절 | 적/아군 | 다음 행동 1회 취소 |
| 보호 | 아군 | 다음 피해 감소 또는 방어도 유지 |
| 반격 | 아군 | 적에게 피해를 받으면 공격 이벤트 생성 |
| 강화 | 아군 | 특정 카드나 필살기 조건에 사용되는 누적 버프 |

## 큐 기반 전투에서의 중요 규칙

### 1. 상태이상은 이벤트 실행 시점에 판정한다

카드를 사용할 때가 아니라, 행동 큐에서 해당 이벤트가 실행되는 순간의 상태를 기준으로 계산한다.

예시:

```text
1. 표식 부여 카드 사용
2. 공격 카드 사용
3. 턴 종료
4. 표식 적용
5. 공격 이벤트가 표식 추가 피해를 받음
```

### 2. 사망한 대상은 기본적으로 이벤트 대상에서 제외한다

이벤트 실행 전에 대상이 사망했다면, 카드별 규칙에 따라 아래 중 하나를 선택한다.

| 규칙 | 설명 |
| --- | --- |
| Fizzle | 이벤트가 무효화된다 |
| Retarget | 조건에 맞는 다른 대상에게 재지정된다 |
| Convert | 방어, 드로우 등 다른 효과로 변환된다 |

MVP 기본값은 `Fizzle`이다. 자동 재지정은 편의성이 좋지만 예측성을 떨어뜨릴 수 있다.

### 3. 지속시간은 턴 정리 단계에서 감소한다

대부분의 상태이상 지속시간은 해당 진영의 턴 정리 단계에서 감소한다.

### 4. 필살기 조건 스택은 일반 버프와 구분한다

필살기 조건에 쓰이는 스택은 일반 전투 버프와 같은 이름을 공유할 수 있지만, 효과와 조건 데이터를 분리해서 관리한다.

예시:

```text
강화
- 전투 효과: 다음 공격 피해 증가
- 필살기 조건: 검사 강화 참격 발동 카운트
```

필살기 조건 스택은 스테이지 내부에서는 전투와 노드를 넘어 유지된다. 스테이지를 클리어하면 기본적으로 초기화된다.

필살기 발동 후 조건 스택은 카드 설계에 따라 아래 중 하나로 처리한다.

| 처리 방식 | 설명 |
| --- | --- |
| Consume | 필살기 발동 후 스택을 소비한다 |
| Preserve | 필살기 발동 후에도 스택을 유지한다 |
| Decay | 턴 종료 또는 피격 시 스택이 감소한다 |

초기 기본안은 `Consume`이다. 필살기 반복 발동을 막고, 플레이어가 같은 스테이지 안에서 다시 준비하는 리듬을 만들기 쉽다.

## 스테이지 초기화 규칙

스테이지 클리어 시 아래 상태는 초기화한다.

- 필살기 조건 스택
- 협동 기믹 진행 카운트
- 스테이지 한정 버프/디버프
- 전투 로그 기반 조건 카운트

일반 카드 보상, 덱 변화, 유물, 장비, 영구 성장 요소는 초기화하지 않는다.

## 우선순위

같은 타이밍에 여러 효과가 발생하면 아래 순서를 따른다.

1. 사망 및 행동 불가 상태 확인
2. 방어/피해 감소 효과
3. 피해 증가/감소 효과
4. 필살기/협동 기믹 조건 확인
5. 피해 적용
6. 피해 후 발동 효과
7. 사망 처리
8. 카드 생성, 드로우, 에너지 회복

## 미정 항목

- 취약/약화 수치를 고정 비율로 할지 스택형으로 할지
- 기절이 보스에게 완전 적용되는지 약화 적용되는지
- 반격이 별도 이벤트로 큐에 들어가는지 즉시 처리되는지
- 표식이 여러 종류로 나뉘는지 단일 상태로 유지되는지
- 필살기 조건 스택이 일반 버프 UI와 같이 보이는지 별도 게이지로 보이는지
