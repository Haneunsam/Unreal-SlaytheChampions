# EnemySystem.md

## 목적

이 문서는 적 데이터, 적 AI, 의도 표시, 일반 몬스터/엘리트/보스 패턴의 기본 구조를 정의한다.

## 적 타입

| 타입 | 역할 |
| --- | --- |
| 일반 몬스터 | 기본 전투 루프 학습 |
| 엘리트 | 덱/파티 빌드 중간 검증 |
| 보스 | 스테이지 전략 검증 |

## 최초 마스터 범위

- 몬스터 5~7종
- 적 최대 3마리 전투
- 엘리트는 유물 확정 보상
- 보스는 전설 카드 확정 보상

## 적 데이터 필드

| 필드 | 설명 |
| --- | --- |
| EnemyId | 적 고유 ID |
| DisplayName | 표시 이름 |
| EnemyType | Normal/Elite/Boss |
| MaxHP | 최대 체력 |
| PatternSet | 행동 패턴 목록 |
| IntentData | 의도 표시 데이터 |
| RewardTag | 보상 풀 태그 |
| SpecialMechanic | 특수 기믹 ID |

## AI 단계

MVP에서는 1~2단계까지만 우선 구현한다.

1. 고정 패턴
2. 랜덤 가중치 패턴
3. 조건부 패턴

## 적 의도 표시

적 의도는 플레이어가 대응 가능한 수준으로 보여준다.

| 의도 | 설명 |
| --- | --- |
| Attack | 피해 예정 |
| Guard | 방어도 획득 |
| Buff | 자기 강화 |
| Debuff | 플레이어 약화 |
| Special | 기믹 행동 |

## 보스 기믹 원칙

- 플레이어의 특정 행동을 유도할 수 있다.
- 단일 정답만 강제하면 안 된다.
- 방어, 공격, 디버프, 회복 등 복수 대응을 허용한다.
- 코스트 감소나 손패 제한처럼 선택 폭을 줄이는 압박은 가능하다.

## 연결 시스템

| 시스템 | 연결 |
| --- | --- |
| `CombatSystem.md` | 적 턴 시작/종료 |
| `EffectResolverSystem.md` | 적 피해/상태 적용 |
| `StatusEffectSystem.md` | 적 버프/디버프 |
| `RewardSystem.md` | 적 타입별 보상 |
| `BattleUISystem.md` | 적 HP/의도 표시 |

## 테스트 기준

- 적이 플레이어 행동 후 행동하는가?
- 적 의도가 UI에 표시되는가?
- 적 패턴이 예측 가능하면서도 반복 피로가 적은가?
- 보스 기믹이 복수 대응 선택지를 남기는가?
- 적 사망 시 전투 종료 조건에 반영되는가?

## 미정 항목

- 적 행동 순서를 좌측->우측으로 할지 속도 스탯으로 할지
- 적 의도 공개 수준
- 보스 페이즈를 MVP에 포함할지 여부
