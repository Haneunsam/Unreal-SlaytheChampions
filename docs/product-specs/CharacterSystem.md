# CharacterSystem.md

## 목적

이 문서는 플레이어 파티원과 직업 구조, 캐릭터별 전투 상태, 파티원 선택 규칙을 정의한다.

캐릭터 시스템은 카드 시스템, 전투 시스템, 상태 시스템과 연결되지만 특정 카드 효과에 직접 의존하지 않는다.

## 핵심 원칙

1. 각 캐릭터는 개별 덱, 손패, draw pile, discard pile을 가진다.
2. 파티는 하나의 공용 코스트를 공유한다.
3. 캐릭터는 자신의 직업 정체성과 카드 태그를 가진다.
4. 전투 중 선택된 캐릭터의 손패만 중심 UI에 표시한다.
5. 캐릭터 사망, 부활, 런 손실 규칙은 추후 확정한다.

## 최초 직업 3종

| 직업 | 전투 역할 | 카드 방향 | 고유 컨셉 |
| --- | --- | --- | --- |
| 검사 | 단일 공격/강화 | 강화 누적 후 강한 공격 | 강화 3스택 후 공격 필살기 |
| 마법사 | 광역 피해/필드 효과 | 원소반응, 필드, 광역 공격 | 마법사 전용 원소반응 |
| 힐러 | 회복/보호/버프 | 회복, 정화, 아군 강화 | 누적 회복/보호 후 대형 지원 |

## 캐릭터 데이터 필드

| 필드 | 설명 |
| --- | --- |
| CharacterId | 캐릭터 고유 ID |
| DisplayName | 표시 이름 |
| ClassId | 검사/마법사/힐러 등 직업 |
| MaxHP | 최대 체력 |
| StartingDeck | 시작 카드 목록 |
| DrawCount | 기본 드로우 수. MVP 기본값 5 |
| ClassTags | 직업/역할 태그 |
| UltimateRuleId | 연결된 필살기 조건 |
| Portrait | UI 초상화 |

## 전투 상태

| 상태 | 범위 | 설명 |
| --- | --- | --- |
| HP | 전투/런 정책에 따라 유지 | 현재 체력 |
| Block | 전투 내부 | 임시 방어도. 기본적으로 턴 정리 시 초기화 |
| Defense Up | 상태이상 지속시간 | 받는 체력 피해를 수치만큼 감소시키는 버프 |
| StatusList | 상태별 Scope | Buff/Debuff/Control/Trigger/Curse |
| Hand | 전투 내부 | 현재 손패 |
| DrawPile | 전투 내부 | 드로우 카드 더미 |
| DiscardPile | 전투 내부 | 사용한 카드 더미 |

## 선택 규칙

- `Q`, `E` 또는 마우스 클릭으로 파티원을 선택한다.
- 선택 파티원의 손패를 중심 UI에 표시한다.
- 비선택 파티원은 HP, Block, 상태, 손패 수, 주요 태그만 요약한다.
- 카드 사용 시 선택된 파티원이 기본 행동 주체가 된다.

## 테스트 기준

- 파티원별 덱과 손패가 분리되는가?
- 공용 코스트를 여러 캐릭터가 공유하는가?
- 선택 파티원의 카드만 사용할 수 있는가?
- 직업 전용 카드가 다른 직업에서 사용되지 않는가?
- 비선택 파티원 정보가 UI에서 요약 가능한가?

## 미정 항목

- 파티원이 전투 중 사망하면 즉시 제외되는지, 부활 가능한지
- HP가 런 전체에서 유지되는지, 전투 후 일부 회복되는지
- 파티원 추가/교체 시스템을 MVP에 포함할지 여부
