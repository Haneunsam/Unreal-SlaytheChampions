# UltimateComboSystem.md

## 목적

이 문서는 필살기 자동 승격과 합동기 입력 패턴 판정 규칙을 정의한다.

필살기는 큰 효과와 연출을 가진 고점 시스템이고, 합동기는 카드 입력 순서로 발동하는 소규모 시너지다.

## 필살기 규칙

| 항목 | 규칙 |
| --- | --- |
| 발동 방식 | 조건을 만족한 카드 이벤트가 실행 시점에 자동 승격 |
| 사용 방식 | 별도 필살기 버튼 없음 |
| 효과 | 기존 카드 효과의 강화판 |
| 연출 | 일반 카드와 구분되는 화면 강조, 이펙트, 사운드 |
| 조건 유지 | 스테이지 내부 유지, 스테이지 클리어 시 초기화 |

## 필살기 조건 데이터

| 필드 | 설명 |
| --- | --- |
| RuleId | 조건 고유 ID |
| ActorClass | 대상 직업 |
| RequiredTags | 필요한 카드/상태 태그 |
| RequiredCount | 필요한 누적 수 |
| RequiredState | 필요한 상태 |
| TriggerEventType | 승격 대상 이벤트 타입 |
| ConsumeRule | 발동 후 스택 소비 여부 |
| UltimateEffect | 강화 효과 |

## 검사 예시

```text
조건: 같은 스테이지 안에서 강화 3스택 보유
발동: 공격 타입 카드 실행
결과: 일반 공격이 검사 필살기 이벤트로 승격
소비: 강화 스택 3개 소비
```

## 합동기 규칙

합동기는 카드 사용 입력 순서가 특정 패턴을 만족할 때 발동한다.

| 패턴 | 효과 예시 |
| --- | --- |
| 공격 -> 공격 -> 공격 | 이번 턴 공격력 소폭 증가 |
| 방어 -> 지원 -> 공격 | 마지막 공격에 소규모 보너스 |
| 공격 -> 지원 -> 공격 | 같은 대상 공격 시 보너스 |

합동기는 기본 카드 이벤트를 대체하지 않는다. 기존 이벤트에 작은 보너스를 부여하거나, 짧은 턴 버프를 추가한다.

## 원소반응 분리

원소반응은 마법사 전용 컨셉이다.

- 합동기 공용 규칙으로 확장하지 않는다.
- 마법사 카드/필드/상태 태그 안에서만 사용한다.
- MVP에서는 원소 태그 1~2개만 사용한다.

## 테스트 기준

- 조건을 만족한 이벤트만 필살기로 승격되는가?
- 필살기 발동 후 스택 소비가 규칙대로 처리되는가?
- 합동기가 입력 패턴 기준으로만 발동되는가?
- 합동기 보너스가 일반 카드 선택을 무의미하게 만들지 않는가?
- 원소반응이 마법사 전용으로 제한되는가?

## 미정 항목

- 합동기 패턴을 몇 개까지 MVP에 넣을지
- 필살기 연출 최소 기준의 실제 구현 방식
