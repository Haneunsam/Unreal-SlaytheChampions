# StatusEffectSystem.md

## 목적

이 문서는 버프, 디버프, 상태이상, 필살기 조건 스택의 데이터와 적용 규칙을 정의한다.

## 상태 분류

| 분류 | 설명 | 예시 |
| --- | --- | --- |
| Buff | 아군에게 이로운 효과 | 힘, 보호, 방어력 증가 |
| Debuff | 대상에게 불리한 효과 | 취약, 약화, 표식 |
| Control | 행동 제한 | 기절, 속박 |
| Trigger | 조건 충족 시 발동 | 반격, 연쇄 |
| Curse | 덱이나 행동 방해 | 부상, 혼란 |
| Ultimate Stack | 필살기 조건 누적 | 강화, 집중, 분노 |

## 기본 상태 후보

| 이름 | 대상 | 효과 |
| --- | --- | --- |
| 취약 | 적/아군 | 받는 피해 증가 |
| 약화 | 적/아군 | 주는 피해 감소 |
| 표식 | 적 | 특정 카드 추가 효과 조건 |
| 기절 | 적/아군 | 다음 행동 1회 취소 |
| 보호 | 아군 | 다음 피해 감소 또는 방어도 유지 |
| 방어력 증가 | 아군 | 받는 체력 피해를 수치만큼 감소 |
| 반격 | 아군 | 피격 시 공격 이벤트 생성 |
| 강화 | 아군 | 필살기 조건 또는 공격 강화 |

## Scope

| Scope | 유지 범위 |
| --- | --- |
| Turn | 현재 턴 |
| Combat | 현재 전투 |
| Stage | 현재 스테이지 |
| Run | 현재 런 |

필살기 조건 스택은 기본적으로 `Stage` Scope를 사용한다. 스테이지 클리어 시 초기화된다.

## 적용 타이밍

```text
이벤트 실행 시작
-> 선처리 상태 확인
-> 주요 효과 처리
-> 후처리 트리거 확인
-> 지속시간 갱신
```

## 테스트 기준

- 상태가 올바른 대상에게 적용되는가?
- 지속시간이 지정된 타이밍에 감소하는가?
- Stage Scope 상태가 스테이지 클리어 시 초기화되는가?
- Defense Up이 피해 계산에 반영되는가?
- 필살기 조건 스택과 일반 버프 효과가 분리되는가?

## 미정 항목

- 취약/약화 수치를 고정값으로 할지 비율로 할지
- 보스에게 기절이 완전 적용되는지 약화 적용되는지
- 필살기 조건 스택 UI를 일반 버프와 분리할지 여부
