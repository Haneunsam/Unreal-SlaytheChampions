# StageGridSystem.md

## 목적

이 문서는 스테이지 그리드 진행, 노드 타입, 스테이지 내부 상태 유지/초기화 규칙을 정의한다.

## 스테이지 흐름

```text
스테이지 시작
-> 그리드 생성
-> 노드 선택
-> 노드 결과 실행
-> 보상/상태 반영
-> 다음 노드 선택
-> 보스 또는 종료 노드
-> 스테이지 클리어
-> 스테이지 한정 상태 초기화
```

## 노드 타입

| 노드 | 역할 |
| --- | --- |
| Battle | 일반 전투 |
| Elite | 고난도 전투, 유물 확정 |
| Boss | 스테이지 최종 검증, 전설 카드 확정 |
| Event | 선택지/특수 보상/저주 |
| Rest | 회복 또는 덱 정비 |
| Shop | 골드 사용 |

## 유지/초기화 규칙

| 항목 | 유지 범위 |
| --- | --- |
| 파티 구성 | 런 전체 |
| 덱 변화 | 런 전체 |
| 골드 | 런 전체 |
| 유물 | 런 전체 |
| 필살기 조건 스택 | 스테이지 내부 |
| 합동기 진행 카운트 | 전투 내부 또는 턴 내부 |
| 손패/Block | 전투 내부 |

스테이지 클리어 시 필살기 조건 스택과 스테이지 한정 상태는 초기화한다.

## 그리드 생성

MVP에서는 고정형 그리드를 우선한다.

- 무작위 그리드 생성은 후순위
- 최소 1개의 보스 노드 포함
- 엘리트, 휴식, 상점 노드는 시간에 따라 축소 가능

## 연결 시스템

| 시스템 | 연결 |
| --- | --- |
| `RunSystem.md` | 현재 Act/스테이지 상태 |
| `CombatSystem.md` | Battle/Elite/Boss 노드 실행 |
| `RewardSystem.md` | 노드 결과 보상 |
| `StatusEffectSystem.md` | 스테이지 클리어 시 상태 초기화 |
| `BattleUISystem.md` | 그리드 화면 UI |

## 테스트 기준

- 선택 가능한 다음 노드만 선택되는가?
- 노드 타입에 맞는 결과가 실행되는가?
- 전투 후 보상이 다음 진행에 반영되는가?
- 스테이지 클리어 시 Stage Scope 상태가 초기화되는가?
- 고정형 그리드가 Act 1 흐름을 만들 수 있는가?
