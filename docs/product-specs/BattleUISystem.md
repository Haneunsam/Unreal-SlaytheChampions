# BattleUISystem.md

## 목적

이 문서는 전투 화면 UI, 카드 사용 화면, 파티원 선택, 행동 큐 표시, 카메라 레퍼런스 적용 기준을 정의한다.

## UI 필수 요소

| 요소 | 설명 |
| --- | --- |
| 파티원 상태 패널 | HP, Block, 상태, 필살기 조건 진행도 |
| 선택 파티원 표시 | 현재 조작 대상 |
| 선택 파티원 손패 | 가장 크게 표시되는 카드 영역 |
| 비선택 파티원 요약 | 손패 수, 주요 태그, 사용 가능 코스트 |
| 공용 코스트 | 항상 확인 가능한 위치 |
| 적 HP/의도 | 적 행동 예측 정보 |
| 행동 큐 미리보기 | 카드 사용 순서와 실행 예정 행동 |
| 턴 종료 버튼 | 플레이어 입력 종료 |

## 조작

| 입력 | 기능 |
| --- | --- |
| Q/E | 이전/다음 파티원 선택 |
| 마우스 클릭 | 파티원 선택, 카드 선택, 대상 선택 |
| Space 또는 버튼 | 턴 종료 |
| Esc | 취소/메뉴 |

## 카드 사용 화면 기준

카드 사용 화면은 `docs/references/visuals/combat-camera-card-use.md`를 따른다.

핵심 방향:

- 3인칭 전투 공간이 유지되어야 한다.
- 선택 캐릭터는 좌측/하단 전경에 둔다.
- 적은 중앙~우측 중경에 둔다.
- 카드 UI는 화면을 완전히 덮지 않는다.
- 행동 큐와 공용 코스트가 항상 읽혀야 한다.

## UI 우선순위

1. 현재 선택 파티원 손패
2. 공용 코스트
3. 행동 큐 실행 순서
4. 적 의도
5. 파티원 HP/Block/상태
6. 필살기 조건 진행도
7. 합동기 패턴 진행도

## 테스트 기준

- 선택 파티원의 손패가 명확히 보이는가?
- 비선택 파티원 정보가 과하게 복잡하지 않은가?
- 카드 사용 순서가 행동 큐에서 확인되는가?
- 적 의도와 보스 기믹이 가려지지 않는가?
- 16:9 기준에서 주요 UI가 겹치지 않는가?

## 미정 항목

- 모바일/패드 대응 여부
- 필살기 조건 UI 위치
- 손패 카드 최대 표시 방식
