# core-loop.md

## 게임 콘셉트

이 게임은 `Slay the Spire` 계열의 로그라이크 덱빌딩 구조를 기반으로 하되, 단일 주인공이 아니라 `1~N명의 파티원`이 카드 이벤트에 따라 순차적으로 행동하는 전투를 핵심 차별점으로 삼는다.

플레이어는 매 턴 각 파티원의 개별 덱에서 나온 카드를 사용해 행동 이벤트를 만든다. 전체 파티는 하나의 공용 코스트 풀을 공유하며, 플레이어는 파티원을 선택한 뒤 해당 파티원의 카드를 사용한다.

턴 종료 시 즉시 결과가 모두 처리되는 것이 아니라, 사용한 카드의 순서와 대상에 따라 파티원의 행동 큐가 실행된다.

예시:

```text
수호자 선택 -> 방어 카드 사용 -> 척후병 선택 -> 공격 카드 사용 -> 턴 종료
=> 수호자가 방어 행동 수행 -> 이후 척후병이 공격 행동 수행
=> 플레이어 행동 종료 후 적 AI 행동 시작
```

## 핵심 루프

```text
스테이지 시작
-> 그리드 노드 선택
-> 전투 진입
-> 카드 선택 및 사용
-> 파티 행동 큐 실행
-> 적 AI 행동 실행
-> 전투 보상 획득
-> 스테이지 내부 상태 유지
-> 덱/파티/유물 성장
-> 다음 노드 선택
-> 스테이지 클리어
-> 스테이지 내부 상태 초기화
```

## 전투 턴 루프

### 1. 턴 시작

- 각 파티원은 자신의 덱에서 정해진 장수의 카드를 드로우한다.
- 파티 전체가 공유하는 코스트가 회복된다.
- 파티원별 상태, 방어도, 지속 효과를 갱신한다.
- 적은 이번 턴 의도 또는 AI 상태를 표시한다.

### 2. 카드 사용

- 플레이어는 파티원을 선택한 뒤 해당 파티원의 손패를 확인한다.
- 파티원 선택은 키보드 `Q`, `E` 또는 마우스 클릭 등으로 수행하는 것을 기본 UX로 검토한다.
- 플레이어는 공용 코스트를 소비해 선택한 파티원의 카드를 사용한다.
- 카드는 즉시 피해를 주기보다 `행동 이벤트`를 생성하는 것을 기본으로 한다.
- 행동 이벤트는 사용 순서대로 플레이어 측 큐에 쌓인다.
- 카드에 따라 자기 자신, 지정 아군, 아군 전체, 지정 적, 적 전체, 모든 유닛, 필드 등을 대상으로 지정할 수 있다.
- 특정 카드 조합, 상태 누적, 사용 순서를 만족하면 협동 기믹 또는 필살기 조건이 충족될 수 있다.

### 3. 턴 종료

- 플레이어가 턴 종료를 누르면 더 이상 카드를 사용할 수 없다.
- 플레이어 측 행동 큐가 앞에서부터 실행된다.
- 각 행동은 지정된 파티원과 대상에 따라 애니메이션, 피해, 방어, 상태이상, 버프를 처리한다.
- 필살기 조건이 충족된 이벤트는 행동 큐 실행 중 자동으로 필살기 이벤트로 승격되며, 일반 카드보다 강화된 효과와 별도 연출을 가진다.

### 4. 적 행동

- 플레이어 측 행동 큐가 모두 끝나면 적 AI가 행동한다.
- 적은 각자의 AI 규칙에 따라 공격, 방어, 버프, 디버프, 소환, 특수 패턴을 실행한다.
- 적도 여러 마리일 수 있으며, 행동 순서는 기본적으로 좌측에서 우측 또는 속도 순서를 사용한다.

### 5. 턴 정리

- 일시 방어도, 턴 제한 버프, 상태이상 지속시간을 정리한다.
- 생존 여부와 전투 종료 조건을 확인한다.
- 전투가 끝나지 않았다면 다음 턴으로 넘어간다.

## 기본 전투 구성

| 요소 | 기본값 |
| --- | --- |
| 플레이어 측 | 1~N명의 파티원 |
| 적 측 | 1~N마리의 적 |
| 덱 구조 | 파티원별 개별 덱 |
| 코스트 구조 | 파티 전체 공용 코스트 |
| 플레이어 입력 | 파티원 선택, 카드 사용 순서, 대상 선택 |
| 플레이어 행동 | 카드가 만든 이벤트 큐를 턴 종료 후 순차 실행 |
| 적 행동 | 플레이어 행동 종료 후 각 적 AI가 실행 |
| 방어도 | 파티원별 개별 방어도 |
| 협동 기믹 | 여러 파티원의 카드 조합, 대상 공유, 순서 조건으로 발동 |
| 필살기 | 특정 상황 조건을 만족하면 기존 카드 이벤트가 자동으로 승격되는 특수 행동 |
| 필살기 조건 스택 | 스테이지 내부에서는 유지, 스테이지 클리어 시 초기화 |
| 핵심 판단 | 어떤 파티원의 카드를 쓸지, 카드 사용 순서, 대상, 코스트 배분, 다음 적 행동 예측 |

## 핵심 의사결정

플레이어는 매 턴 다음 질문에 답해야 한다.

- 이번 턴에 방어를 먼저 쌓을 것인가, 공격으로 적을 줄일 것인가?
- 공용 코스트를 어떤 파티원의 카드에 배분할 것인가?
- 어떤 파티원의 손패가 이번 턴 적 의도에 가장 잘 대응하는가?
- 이번 턴에 협동 기믹 또는 필살기 조건을 노릴 것인가?
- 필살기 조건을 쌓기 위해 현재 턴의 즉시 효율을 포기할 가치가 있는가?
- 적의 다음 행동을 보고 피해를 감수할지, 방어/제어로 넘길지 결정할 것인가?
- 현재 덱을 강화할 카드를 고를 것인가, 덱을 얇게 유지할 것인가?
- 파티 조합과 카드 시너지를 어떤 방향으로 성장시킬 것인가?

## 현재 확정된 방향

- 장르: 로그라이크 덱빌딩
- 전투: 파티 기반 카드 이벤트 큐 전투
- 행동 순서: 플레이어 카드 이벤트 큐 실행 후 적 AI 실행
- 파티 규모: 1~N명
- 적 규모: 1~N마리
- 덱 구조: 파티원별 개별 덱
- 코스트 구조: 파티 전체 공용 코스트
- 카드 사용 UX: 파티원 선택 후 해당 파티원의 카드 사용
- 카드 기본 분류: 공용 카드, 직업 전용 카드, 저주 카드, 특수 카드
- 카드 대상: 자기 자신, 아군 전체, 아군 지정, 적 지정, 적군 전체, 모든 유닛 전체, 필드 효과
- 방어도: 파티원 개별 적용
- 방어 예외: 카드 효과로 피해 대신 받기, 모든 파티원 방어도 증가 가능
- 협동 기믹: 파티원 간 카드 조합, 순서, 대상 공유로 발동
- 필살기: 특정 조건을 만족하면 행동 큐 실행 중 자동 승격되며, 큰 데미지, 큰 버프, 큰 디버프 등으로 기존 카드 효과를 강화하고 별도 연출 적용
- 필살기 조건 스택: 같은 스테이지 안에서는 전투/노드를 넘어 유지, 스테이지 클리어 시 초기화
- 핵심 재미: 파티원 선택, 코스트 배분, 카드 순서 설계, 협동 기믹, 필살기 준비, 파티 시너지, 위험 예측

## 미정 항목

- 적의 의도를 완전히 공개할지, 일부만 공개할지
- 파티원이 사망하면 전투 중 부활 가능한지, 런 전체에서 손실되는지
- 파티원별 드로우 장수를 고정할지, 파티원마다 다르게 둘지
- 전투 UI에서 비선택 파티원의 손패를 얼마나 노출할지
